What is jQuery?

jQuery is a JavaScript library. The primary usage of jQuery is to make your HTML pages more responsive and interactive with less code. It is widely supported by desktop and platform browsers. Since the initial release of jQuery 1.0 back in June 2016, we have seen lots of updates and additions. This chapter will teach you the basics of jQuery. For a comprehensive list of jQuery's usage, you should visit jquery.com. What you should remember is jQuery is very good at making HTML elements dynamic and fun.

How does jQuery work?

Before we dive into the usage, let's first see an example by CodeAcademy. On the Result tab, try hover the mouse over, away or click on the color blocks to see what happens.

On the JavaScript block, you probably immediately noticed $(document) and $('.div'). Inside of the $(), we put in selectors (similar to selectors in CSS). Then it is followed by function calls .ready(), .mouseenter(), .mouseleave(), .click(). Note that higher-order functions are then passed in as arguments. We will continuously see/use them. If you are not comfortable with higher-order functions, I recommend you watching this video series on Youtube here.

Still confused? It's ok. Now we are going to look at them piece by piece.

Getting Started

In the above example, the first line goes $(document).ready(some function).

  • $() is an alias for the function called jQuery. It announces that "we are using jQuery here!"
  • document is the document object in HTML DOM, containing information of the entire HTML document. It is the "thing" that the function will have effect on.
  • .ready() is a function defined in the jQuery library. It tells that "something is going to happen after the HTML document is fully loaded."
  • some function inside of the paranthese of .ready() is going to be effective as soon as the page is ready. It usually looks like this - function(){}. In our above example, it contains the methods that change the shape's height when mouse hovers on the object and fade the object away when mouse is clicked.

Most of the time, you want to have $(document).ready() at the beginning of your jQuery magic. Becausee it is unsafe to manipulate a safe until it is fully loaded. Although your code might work without it, you should not do so.

More jQuery Syntax

Now let's look inside of the function to be execute after the HTML is ready.

  • $('div').mouseenter(some function) has a very similar structure as $(document).ready().
  • $('div') selects all <div> elements on the HTML. In the example above, it points to the four <div> elemnts, with the id "red", "blue", "yellow" and "green". Note that it has quotes around it. In fact, other than document, all other selectors need to be sorrounded by quotes.
  • .mouseenter(some function) is another defined function in jQuery library. It tells that "something is going to happen after the mouse enters into the selected element."

It should be easy enough to guess what the other functions do. $('div').mouseleave(some function) does something when the mouse leaves the <div> elements; $('div').click(some function) gets triggered when the mouse is clicked.

Looking further inside of the function, you see the magic word of this inside of $(). Its usage is similar to this in many other programming languages. Only that you will need to wrap it inside of $() when you are calling a jQuery function on the object. .animate() and .toggle() are both jQuery functions, so you use $(this) instead of this. I'm sure you have guessed how .animate() and .toggle() work. To see more details, check out the jQuery documentation.

One Last Thing

jQuery is not a standard JavaScript library. To use it, you need to provide your browser with access to jQuery library first.

On your HTML, simply include the code below.

<script src="https://code.jquery.com/jquery-1.12.3.min.js" type="application/javascript"></script>
