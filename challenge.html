Making "Nested" API Calls

In this final challenge, you will try to conquer a more complicated situation - making two consecutive API calls, with the second call based on the first call's response.

Are you ready?

Recall that in one of the previous chapters, you did a challenge to create a functional API call:

Obviously, this is not very visual, because the images are not displayed as images but text. Click on the link and see where it lands you.

Did you notice that the links are not going into the actually images. Instead, it's a page that contains the image's information. In fact, the image's direct URL is not returned in the API call we've made.

In this final challenge, you will obtain the image's direct URL by making a second API call. The second API call will be based on the response from the first API call. Your result should be showing 10 actual images on the page. You may apply any CSS style as you prefer.

Hint:

  • using recursive functions can help you call the same function multiple times.
  • to get the image URL, you will need to use the action=query again in the second API call by telling the server the related image name. Read more here.
  • if you have trouble structuring the html code, try to do it without variables first.
