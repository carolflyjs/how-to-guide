Enrich Your Website with API

In the previous chapters, we learned about obtaining API data. But what do you do with them? Surely you don't want to just console.log() everything, because visitors do not look into the console (and it's formatted somewhat ugly). Therefore, you need to be able to parse the received data and push them to the HTML.

Populating HTML with jQuery

In an early chapter, we used this code to explain JSON data strcture. The code on the last line is HTML population in its simpliest form.

$('.output').html(ashley_location);

.html(something) is a jQuery function that sets the selected object in the document with something. This something is expected to be a string formatted in html.

Let's make it a little fancier.

Instead of pushing the plain ashley_location, we pushed "<em>" + ashley_location + "/<em>". What this does is to wrap the original string with a pair of "<em>" tag. We know that ashley_location is a string "Hong Kong". So "<em>" + ashley_location + "/<em>" produces <em>Hong Kong</em>.

It wasn't too complicated, so time to level up! Often, you obtain values in an array like this one below.

This one used a for loop that repetitively modifies the HTML code. Firstly, what exactly is being looped through?

i < data[1].length gives the answer awway. It is looping through the 2nd value in the received object. Because it is using the .length function, you can guess that this 2nd value is an array.

["Coffee", "Coffeehouse", "Coffee in Seattle", "Coffee preparation", "Coffeeshop Company", "Coffee bean", "Coffee production in India", "Coffee and doughnuts", "Coffee production in Indonesia", "Coffee production in Colombia"]

This array contains all the page names returned from the search. There is a total of 10 results returned, and therefore the loop stops after the 10th iteration.

In each iteration, it produces a chunk of html code, and then .append() it to the target object - the <div> tag. .append(something) is a function used to add new content after existing content. It works similarly as the += operator in programming languages.

The rest of the trick was pulled by the following code, which looks more complicated than it actually is: "<div><h2><a href=" + data[3][i] + ">" + data[1][i] + "</a></h2>" + "<p>" + data[2][i] + "</p></div>"

data[1][i] = page name, data[2][i] = page description, and data[3][i] = page url. Essentially, what we put together is <div><h2><a href={{page url}}">{{page name}}</a></h2><p>{{page description}}</p></div>".

So far so good? In this chapter, we learned about how to parse data into html. Together with the API request making, we can make things more fun. Ready for the final challenge?
