What is JSON?

JSON = JavaScript Object Notation.

If you have worked with objects in JavaScript, you already know that "object" is a data type that contains properties and methods, just like objects in real-life. JSON is specifically designed for data exchange. In Web APIs, JSON is one of the most popular formats for applications to exchange information. In this chapter, you will learn how to work with JSON properties through comparing it to JavaScript.

How to work with JSON?

First, this is a comparison of JSON and JavaScript string object:

JSON JavaScript

The data in the two are equivalent, with both objects being nested and assigned to var friends. One of the main differences is the use of quotes around properties. In JSON, all properties need to be in quotes. In regular JavaScript object, there's no such restrictions unless the property name contains spaces. Still, the structures don't look very different.

As it turns out, JSON is a standard built-in object in JavaScript these days, so even accessing the JSON object properties is similar to a JavaScript object.

JSON JavaScript

For both JSON and JavaScript objects, we used friends.ashley["current location"] to access ashley's current location. The dot notation is used when the property name doesn't contain any space, and the bracket notation is used when space is present. If you don't understand what $('.output').html(ashley_location) is, don't worry. We'll explain it later in the jQuery chapter.

So far so good? Now let's look at one last thing before moving on: converting JavaScript object to JSON format.

As you see, we used a built-in JSON function JSON.stringify(friends) to convert the JSON object to a JavaScript string object. As a result, all property names are now wrapped around by quotes. Again, JSON is a built-in object, and stringify() is a native method.

There is another frequently-used method JSON.parse(data) which is used to process JSON data. The data argument passed into the function is often the .json file received through API. You will see examples processing JSON data in a later chapter Display Data in HTML. For now, let's not worry too much about it and continue our planned journey.
